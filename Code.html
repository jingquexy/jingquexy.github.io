Code

GaitSet, SMPLGait, GaitPart, GaitGL代表了步态识别领域中几种截然不同的技术路线。基于代码文件，从核心理念、输入数据、时序建模、空间建模四个维度为详细剖析它们的差异。

1. 核心理念与架构 (Core Philosophy)

  • GaitSet (基于集合的 2D 提取)

    • Set：认为步态序列不需要严格的时间顺序，可以将其视为一组无序的图像集合。
    • 使用 SetBlockWrapper 和 Set Pooling (Max Pooling) 将时间维度直接压缩，强调“置换不变性”。
    • 解决输入帧数不固定、帧序混乱的问题，追求推理速度。

  • SMPLGait (3D 先验辅助)

    • **Prior：**2D 轮廓图受视角影响大，利用 3D SMPL 人体参数模型作为“先验知识”，显式地校正和对齐视觉特征。
    • 引入了一个 MLP 分支处理 SMPL 参数，生成变换矩阵，并通过 torch.bmm 作用于视觉特征。
    • 解决跨视角（Cross-View）识别难、姿态变化大的问题。

  • GaitPart (局部微动捕捉)

    • Micro-motion：人体不同部位（如头、手、腿）的运动模式完全不同，应该分块独立建模，且关注短时的微动特征。
    • 使用 TemporalFeatureAggregator (TFA) 和 1D 卷积捕捉细节，且不同部位的卷积权重不共享。
    • 解决细粒度识别问题，区分走路姿势非常相似的人。

  • GaitGL (3D CNN 时空融合)

    • **Spatiotemporal：**利用 3D 卷积同时提取空间和时间特征，并在每一层卷积内部融合全局和局部信息。
    • 使用 BasicConv3d 和 GLConv 模块，包含全局分支和局部分支的融合机制。
    • 追求 SOTA，捕捉最丰富的时空特征。

2. 输入数据 (Input Modality)

这是区分 SMPLGait 和其他模型的最直观特征。

  • GaitSet, GaitPart, GaitGL:

    • 输入：仅输入 2D 轮廓图序列 (Silhouettes)。sils = ipts[0]。
    • 属于端到端的纯视觉方案，部署更简单。

  • SMPLGait:

    • 输入：多模态。输入 2D 轮廓图 + 3D SMPL 参数 (85维: Pose+Shape+Trans)。sils = ipts[0], smpls = ipts[1]。
    • 意味着即使在测试阶段，也必须先运行 3D 人体姿态估计算法（如 HMR 或 SPIN）获取参数。

4. 空间与局部建模 (Spatial & Part Modeling)

如何处理人体的不同部位（头、躯干、腿）？区分了模型是“整体处理”还是“分块处理”，以及“分块”发生的时机。

  • GaitSet (全局流 + 后期切分)

    • 主要依赖 CNN 提取全图特征。
    • 虽然有 MGL (Multilayer Global Pipeline) 融合多层特征，但对“部位”的切分（HPP）仅发生在网络的最末端。

  • SMPLGait (3D 矩阵校正)

    • **空间对齐：**它不直接切分部位，而是通过 SMPL 参数生成一个16×1616 \times 1616×16的 变换矩阵。
    • 通过 torch.bmm(outs, sps)，利用 3D 信息对 2D 特征图的像素进行加权和对齐。

  • GaitPart (独立权重切分)

    • 关键差异：它在网络中间就进行了 HPP 切分 out = self.HPP(out)。
    • 独立性：对于切分后的 PPP个部位，使用了 clones(conv3x1, parts_num) 创建了互不共享权重的独立卷积网络。这意味着头部和腿部是完全不同的两个子网络在学习。

  • GaitGL (层内融合 + 共享权重)

    • GLConv：在每一层卷积层内部，将特征图物理切分为局部块 (x.split)，同时也保留全局块。
    • 共享性：与 GaitPart 不同，GaitGL 的局部分支使用的是共享权重的卷积核 (self.local_conv3d)。
    • GeMHPP：使用了可学习的广义平均池化 (GeM)，而不是固定的 Max/Avg Pooling。

SkeletonGait++

论文的核心突破在于解决了两个技术难题：输入端如何让 CNN 看懂稀疏的骨骼点，以及融合端如何让网络在骨骼和轮廓之间做选择。

1. Skeleton Maps 骨架图生成：从“点”到“图”的转化 (输入端)

传统 CNN 擅长处理密集的网格数据（如图像），而骨骼数据通常是稀疏的坐标点 (x,y)(x, y)(x,y)  。为了让 DeepGaitV2 这种强大的 CNN 架构能处理骨骼数据，论文提出了一种 “栅格化渲染” 方案，将坐标点转化为类似热图（Heatmap）的 Skeleton Map。

这个过程包含以下关键步骤：

第一步：坐标归一化 (Normalization)

原始的骨骼坐标包含了很多干扰信息（如拍摄距离远近、人在画面中的行走轨迹）。为了提取纯净的步态特征，首先要进行归一化：

  • 中心归一化 (Center-normalization)：计算左右髋关节的中心作为人体质心，将所有关节点减去该质心坐标，把人“移”到画面中心。

  • 尺度归一化 (Scale-normalization)：将人体高度统一缩放为固定高度 HHH。

    • 目的：消除相机距离和人物身高的影响，专注于肢体的相对运动。

第二步：高斯渲染 (Gaussian Rendering)

将归一化后的坐标点渲染为图像。论文生成了两张图，分别对应两个通道：

[image]

  1. 关节图 (Joint Map, JJJ )：

    • 将每个关节点 (xk,yk)(x_k, y_k)(xk​,yk​)看 作一个高斯分布的中心。
    • 公式：J(i,j)=∑ke−(i−xk)2+(j−yk)22σ2×ckJ_{(i,j)} = \sum_{k} e^{-\frac{(i-x_k)^2 + (j-y_k)^2}{2\sigma^2}} \times c_kJ(i,j)​=∑k​e−2σ2(i−xk​)2+(j−yk​)2​×ck​。
    • 直观理解：关节点处最亮，向四周晕染，形成光点。

  2. 肢体图 (Limb Map, LLL )：

    • 将相邻关节点连线，计算像素点到线段的距离 D\mathcal{D}D 。
    • 公式：L(i,j)=∑ne−D((i,j),Sn)22σ2L_{(i,j)} = \sum_{n} e^{-\frac{\mathcal{D}((i,j), \mathcal{S}_n)^2}{2\sigma^2}}L(i,j)​=∑n​e−2σ2D((i,j),Sn​)2​ 。
    • 直观理解：肢体骨骼连线处最亮，形成光棒。

第三步：堆叠与裁剪 (Stacking & Cropping)

  • 双通道输入：将 JJJ 和 LLL 在通道维度堆叠，形成 2×H×W2 \times H \times W2×H×W的张量。
  • 中心裁剪：为了减少图像中的空白冗余，进行基于主体的裁剪（Subject-centered cropping），最终调整为 64×4464 \times 4464×44的 大小，这与轮廓图（Silhouette）的输入尺寸完全对齐。

[image]

技术总结：它生成了像轮廓图一样的网格图像（CNN可以处理），但完全剔除了体型、衣着等外观信息，只保留了纯粹的骨架结构。

2. Attention Fusion 注意力融合 (融合端)：自适应加权机制

在 SkeletonGait++ 中，网络拥有两条分支：一条看轮廓（Silhouette），一条看骨架（Skeleton）。Attention Fusion 模块的作用是根据当前的图像质量，自动决定“听谁的”。

[image]

该模块的代码实现位于 AttentionFusion 类中，其核心是一个轻量级的卷积网络，结构如下：

网络结构：1x1→3x3→1x1

这是一个经典的“瓶颈”（Bottleneck）结构，旨在以最小的计算代价捕捉局部上下文信息。

  1. 输入拼接 (Concat)：

    • 代码：feats = torch.cat([sil_feat, map_feat], dim=1)
    • 将轮廓特征和骨架特征在通道维度拼接，输入通道数变为 2C2C2C 。

  2. 压缩 (Squeeze, 1×11 \times 11×1C onv)：

    • 代码：conv1x1(in_channels * 2, hidden_dim)
    • 作用：降维。将通道数从 2C2C2C压 缩到 hidden_dim（通常是 1/161/161/16 ），减少计算量，强迫网络提取最核心的信息。

  3. 感知上下文 (Context, 3×33 \times 33×3C onv)：

    • 代码：conv3x3(hidden_dim, hidden_dim)
    • 作用：引入空间感受野。网络不仅看当前像素，还参考周围像素的信息来判断当前区域的质量（例如，如果是边缘模糊区域，可能轮廓特征不可靠）。

  4. 扩张 (Excitation, 1×11 \times 11×1 Conv)：

    • 代码：conv1x1(hidden_dim, in_channels * 2)
    • 作用：升维。将通道数恢复到 2C2C2C ，为生成注意力分数做准备。

生成掩码与加权 (Mask Generation & Weighting)

经过上述网络后，得到的是未经归一化的分数（Score）。接下来的步骤将分数转化为概率掩码：

  1. Softmax 归一化：

    • 代码：score = F.softmax(score, dim=1) 。
    • 逻辑：将通道分为两组（一组对应轮廓，一组对应骨架）。对这两组分数进行 Softmax，确保对于每个像素点 (x,y)(x,y)(x,y) 和每个通道 ccc ，都有 Weightsil+Weightskel=1Weight_{sil} + Weight_{skel} = 1Weightsil​+Weightskel​=1 。
    • 直观理解：如果网络认为这一帧的轮廓因为遮挡烂掉了，就会把权重全部拨给骨架。

  2. 加权求和 (Weighted Sum)：

    • 代码：retun = sil_feat * score[:, 0] + map_feat * score[:, 1]。
    • 结果：输出融合后的特征，它动态地结合了两个分支的优点。

技术总结：这个模块之所以有效，是因为它不仅考虑了“像素值”，还通过 3×33 \times 33×3 卷积考虑了“局部结构”。在夜间或遮挡场景下（轮廓缺失），该模块能敏锐地通过 Softmax 机制抑制轮廓分支的噪声，放大骨架分支的信号。
